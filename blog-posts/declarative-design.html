Declarative Design & Web Thinking: Insights from Jeremy Keith

Declarative vs. Imperative: A Musical Lens

Design philosophies mirror musical approaches:

  • Classical music relies on precise sheet music with strict instructions.
  • Jazz thrives on improvisation and minimal rules.

Programming Paradigms

Imperative (JavaScript)

  • Step-by-step control (e.g., create > loop > test > return).
  • Fragile: One error breaks the entire flow.

Declarative (HTML/CSS)

  • Describe outcomes, not steps (e.g., font-size: clamp(...)).
  • Fault-tolerant: Errors are skipped, not displayed.

“JavaScript should only do what only JavaScript can do.”

Modern CSS Techniques

font-size: clamp(1rem, 0.5rem + 0.666vw, 1.5rem);
  • Use calc(), min(), and max() for fluid spacing.
  • Leverage fit-content() and minmax() for grids.

Resources: Utopia.fyi for fluid scaling, Every-Layout.dev for layout patterns.

Design Thinking Approaches

Analytical Thinking

  • Break systems into parts (e.g., chemistry or physics).
  • Focus on zooming in to understand components.

Systems Thinking

  • Study how parts interact holistically.
  • Focus on zooming out to see connections.

Web vs. Native Philosophy

  • Web: Prioritize declarative design (HTML/CSS first).
  • Native apps: Optimize for imperative control.

Essential read: A Dao of Web Design, which challenges assumptions about devices and users.

The web thrives on flexibility. Like jazz, declarative design embraces improvisation—let the browser handle the details.

Leerpunten

Wat ik heb geleerd van Jeremy Keith’s lezing

  • Declaratieve code is flexibeler: Ik laat HTML/CSS meer het werk doen, omdat ze fouten negeren en zich aanpassen – minder micromanagen met JavaScript.
  • Jazz als inspiratie: Net zoals improvisatie in jazz, moet webdesign ruimte bieden voor onverwachte scenario’s (gebruikers én browsers).
  • CSS is een suggestie: Geen pixel-perfecte controle nodig – tools zoals clamp() laten de browser zelf slimme keuzes maken.
  • Systeemdenken: Niet alleen code-analyseren, maar ook kijken hoe alles samenwerkt voor robuustere ontwerpen.
  • Het web is uniek: Geen native app-nabootsing – aanpasbaarheid en veerkracht zijn juist krachten.

Mijn belangrijkste inzicht: Minder controle = betere toegankelijkheid. Ik ga vaker clamp() en Utopia.fyi gebruiken!
