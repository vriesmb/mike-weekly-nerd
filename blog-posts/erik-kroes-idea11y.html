Practical Fixes for common WCAG Failures

What is accessibility

  • Accessible product is a product thats usable by everybody, including people with disabilities
  • Accessibility doesnt mean it's great.. it's just technically useable.

Keyboard testing

  • Pressing Tab - should take you to every interactive element, in an intuitive order, on the website
  • Most common keyboard issues:
  • No skip links (you want to be able to skip the menu)
  • Uncontrolled focus state (visible confirmation - see where you are)
  • <div role="button"> - works with mouse, not with keyboard should use right HTML elements
  • Visible label is better than a placeholder. The moment you need it the most, it disappears.

Contrast rules/tips

  • Contrast helps everybody. When your screen is sunny, you can still see the contrast difference
  • UI elements: minimum 3:1 contrast ratio
  • Large text (24px/19px bold): minimum 3:1
  • All other text: minimum 4.5:1
  • Most common issues:
  • Placeholders - text too low in contrast
  • Branding colors prioritized over accessibility

Alternatives

  • Offer captions in videos
  • Offer transcript for podcasts
  • Images: use alt text (decorative images need no alternative)
  • Most common issues:
  • Image in buttons or links
  • Video content without alternatives

Sizing

  • Compare mobile, tablet and desktop - no screen size is the same
  • Most common issues:
  • Design not made for 320px
  • External content (cookie notice, videos, popups) breaking accessibility
  • Resource: https://www.solidstart.info
