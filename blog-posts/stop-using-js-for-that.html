Notities

[Stop using JS for that - Kilian]

  • Electron framework voor software maken. Polypane browser voor developers. [educcation.github.com](http://educcation.github.com/)
  • JSX /> Rule of least power Choose the least powerful language suitable for a given purpose
  • Er zijn altijd meerdere methodes voor functionaliteiten niet maar 1 versie. Alles op het web werkt altijd.
  • Custom switches Browser doet ook wat werk. Clicks registreert hij van hele labels om een checkbox heen.
  • :checked pseudo selector + ::before (transform translate x (1rem)) voor animatietje. Appearance none = eigen styling dmv ::before en ::after :focus '&' :focus-visible
  • ruleset zijn de hele css conditions binnen de kringelhaken {}. input type ="color”. geeft colorpicker UI
  • **In-page transitions** Scroll-behavior: smooth;
  • **Accessibility** @media.   prefers-reduced-motion: no-preference
  • Progressive enhancement
    #my-traget for scroll Scroll-margin-top 100px
    #my-target:target{ outline transition ease } voor een highlight van target
  • **Accordions and modals**
    Details *open*> (open voor eerste openhebben staan good default - live attribute browser dynamisch toevoegen en weghalen als je klikt)
    summary (standaard zichtbaar)
    p (complete info)
    details />
    Marker element in accordion summary::marker{ fontsize content :’’; [open] summary::marker
    Summary is clickable Summary heeft een :hover nodig want krijgt geen ander cursor anders kan met cursor:pointer of styling change gebruiker moet je melden is interactief
  • **Dialog / Popup** Dialog> form method”dialog” h1 button type”submit” close (want een modal sluit niet zelf, moet aangestuurd worden) /form /dialog moet je tripleren met button onclick “dialog.showModal geeft je popup / dialog Dialog komt op top layer zit niet meer in pagina.
  • **Backdrop** is complete ruimte achter je dialog/popup dialog:backdrop{} dialog{ opacity 1 transform scale1 transition all 1s ease @starting-style{ opacity: 0; transform: scale(0.8); } }
  • **Parent Selector** :has() wil .. stylen alleen als er een .. in zit form:has(#other:checked) #other-text { … } input, textarea { field-sizing: content; } field wordt net zo groot als input content
  • **Select element** Select> option> option> /Select> select, ::picker(select) { appearance: base-text; (ik bepaal styling) }
  • **Scroll driven animations** https://scroll-driven-animations.style/demos/cover-flow/css
