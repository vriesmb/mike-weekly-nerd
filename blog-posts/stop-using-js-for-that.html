Stop using JS for that

In deze talk laat Kilian Valkhof zien dat we véél te vaak grijpen naar JavaScript, terwijl HTML en CSS tegenwoordig zélf de klus klaren. Slimmer, sneller én toegankelijker.

★ ★ ★ ★ ★

Progressive Enhancement first

Kilian benadrukt het belang van de Rule of Least Power: kies altijd de minst krachtige taal die voldoet. Dus HTML boven CSS, CSS boven JS. Hierdoor wordt je site robuuster, sneller en toegankelijker.

Alles op het web werkt altijd – of zou dat moeten. En als iets niet werkt, moet de basis blijven functioneren (progressive enhancement).

Custom UI zonder JavaScript

Je kunt tegenwoordig complete interactieve elementen maken met enkel HTML en CSS:

  • <details> en <summary> voor accordeons
  • <dialog> + .showModal() voor popups
  • :checked pseudo-selector + ::before voor switches
  • <input type="color"> voor een ingebouwde colorpicker
  • :has() als parent selector om dynamisch te stylen

Je hoeft dus niet alles in JavaScript te bouwen – browsers doen tegenwoordig veel werk zelf.

Toegankelijkheid en animaties

Met media queries als @media (prefers-reduced-motion: no-preference) kun je motion toegankelijk maken. Ook kun je elementen laten oplichten bij navigatie met :target en scroll-margin-top gebruiken voor betere focusposities.

Focus styling zoals :focus en :focus-visible is essentieel voor keyboardgebruikers.

Scroll driven animations

Met nieuwe CSS-technieken zoals scroll-driven animations kun je geavanceerde animaties maken die reageren op scrollgedrag – zónder JS.

Tools van Kilian zelf
  • Polypane: een browser speciaal voor developers, met preview van meerdere schermgroottes tegelijk
  • Educcation.github.io: leer CSS, HTML en webstandaarden in kleine interactieve lessen

Reflectie

Er zijn altijd meerdere manieren om een feature te bouwen. JavaScript is krachtig, maar dat betekent niet dat je het overal voor hoeft in te zetten. Door eerst te kijken wat je met HTML en CSS kunt bereiken, houd je je site sneller, toegankelijker en beter te onderhouden.

Gebruik JavaScript bewust en alleen als het echt iets toevoegt.

Sta af en toe stil bij je keuzes: wat heb je écht nodig? Welke oplossing is het meest toekomstbestendig? Een bewuste aanpak levert vaak simpelere, elegantere code op – en daar plukken jij én je gebruikers de vruchten van.
