Leerdoelen

Transition

  • Ik wil vanuit de basis van een project de layout en structuur robuust opmaken zodat de website niet op het laatste moment nog pixel perfect gemaakt hoeft te worden, denk hierbij aan nette responsiveness, consistent sizing etc.. Ik ben tevreden als de componenten op de website netjes zijn opgemaakt. Responsive, herbruikbaar en variabel ingesteld indien toepasselijk.
  • Ik ben in staat nette applicaties te ontwerpen maar ik wil nu gebruikers een prettige saemless (UX) ervaring geven op de applicatie. Dit wil ik gaan doen door de gebruiker echt aan te geven, aan de hand van micro-interacties en animaties, wat er gebeurd op de website.
  • ik wil goed worden in georganiseerd code schrijven om het door te kunnen pasen aan anderen developers. Ik ben tevreden als mijn code duidelijk leesbaar is in de logica of stijling maar ook a.d.v. comments en een goede structurele hierarchie indien toepasselijk.

Persoonlijke reflectie 2/3
Ik heb gemerkt dat ik aan het begin van elk project erg moet inkomen. Elk vak is continu een enorme switch in denkwijze. Het is vaak lastig om ideeen meteen uit te werken.. en dat knaagt wel. Verder ben ik eigenlijk altijd super trots op het eindresultaat, maar de weg er naar toe is wel oneindig. Ik heb veel verschillende technieken geprobeerd qua APIs heb ik zelfs met speech gewerkt, in css met een soort 3D illusie, heb met API content gewerkt (en dat is nog maar een klein deel). Allemaal dingen die ik aan het begin nog niet kon en waarvan ik ook niet wist dat het mogelijk was.
